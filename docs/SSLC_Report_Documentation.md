## 1. Overview

The **SSLC Report** is a comprehensive psychometric PDF report generated for Class 10 students. It combines **DISC personality profiling**, **Agile scrum value scores**, and **academic stream analysis** to deliver a personalized career and academic guidance document.

The report is generated by the `SchoolReport` class which extends `BaseReport`. The `generate()` method orchestrates PDF creation in a fixed pipeline; SSLC-specific content is branched by `school_level_id === 1`.

The SSLC report is unique from the HSC report in that it:
- Includes a **"Where You Fit Best"** section with stream recommendation engine
- Shows **all 6 academic streams** (PCMB, PCB, PCM, PCBZ, Commerce, Humanities) with detail cards and odyssey roadmaps
- Shows **course compatibility for the recommended stream** (not the student's current stream)
- Does **not** include "Reach Institutions" (colleges/university table)

## 2. Input Data Model

The report receives a `SchoolData` object. Key fields:

| Field | Type | Purpose |
|---|---|---|
| `full_name` | `string` | Student's name, used in headings & text |
| `exam_ref_no` | `string` | Shown vertically on cover, used in page footers |
| `report_title` | `string` | Bold title on cover page |
| `exam_start` | `Date` | Formatted as date on cover page |
| `school_level_id` | `1` (SSLC) or `2` (HSC) | Branch selector for SSLC vs HSC sections |
| `school_stream_id` | `number \| null` | Stream ID for HSC only; for SSLC this is null/unused |
| `score_D` | `number` (0–100) | Dominance DISC score |
| `score_I` | `number` (0–100) | Influence DISC score |
| `score_S` | `number` (0–100) | Steadiness DISC score |
| `score_C` | `number` (0–100) | Conscientiousness DISC score |
| `most_answered_answer_type` | `array` of `{ANSWER_TYPE, COUNT}` | Used to determine top-two DISC traits by frequency |
| `agile_scores[0]` | `AgileScore` | Raw agile values: `commitment`, `courage`, `focus`, `openness`, `respect` (each 0–25) |

### 4.4 General Characteristics (Personalized Insights)
**Methods:** `generatePersonalizedInsights()` → calls `ci_generateCorePersonality()`

**Inputs used:** Primary DISC type from `most_answered_answer_type`, student name

**What it generates:**

#### H1: "General Characteristics for [Student Name]"
Two paragraphs from `SCHOOL_DYNAMIC_CONTENT[primaryType].general_characteristics_1` and `_2`.

**4 variations — one per primary DISC type:**

| Type | Opening Framing |
|---|---|
| **D** | "You exhibit a natural drive to take charge and achieve results…" |
| **I** | "You possess a strong confidence in your ability to inspire and motivate others…" |
| **S** | "You are naturally dependable and supportive, with a strong focus on maintaining harmony…" |
| **C** | "You have a natural inclination toward precision, quality, and structure…" |

#### Sub-section: Core Personality Profile
**Method:** `ci_generateCorePersonality()`

Behavior branches on `ci_patterns.discType`:

| `discType` | Condition | Output |
|---|---|---|
| `dominant` | Top trait >75 AND gap >10 from #2 | Renders an "archetype card" with title, superpower, risk, environment. Pulls from `ARCHETYPE_DATA[dominantTrait].dominant` |
| `dual` | Top two both >65 AND gap <5 | Renders title from `DUAL_ARCHETYPE[topTwo]` + two archetype cards (one per trait, from `.secondary`) |
| `balanced` | Low stddev (<8) or gap <5, no dominant | Renders "Versatile Adaptive Profile" with a text variation |

Each archetype card shows:
- ⚡ **Superpower** — the trait's core behavioral strength
- ⚠ **Risk Area** — likely blind spots
- 🌐 **Environment** — best working/study context

Text variations (`ci_tv()`) are picked by `textVariant = (D + I + S + C) % 3`, cycling 3 possible strings per key.

#### H2: "Understanding Yourself - Who I Am"
Two paragraphs per DISC type from `understanding_yourself_1` and `_2`.

### 4.19 Where You Fit Best *(SSLC-exclusive)*
**Method:** `generateWhereYouFitBest()`

**How the recommended stream is determined:**

1. Compute top-two DISC traits → `traitCode` (e.g., `"SD"`)
2. Get student's agile scores and find the **highest raw agile dimension** ("Focus", "Courage", "Respect", "Openness", or "Commitment")
3. Look up each stream's compatibility with that top agile dimension in `STREAM_AGILE_COMPATIBILITY` matrix
4. Rank all 6 streams by compatibility score; **top-ranked stream is recommended**
5. Fetch 3 alternative streams (ranks 2, 3, 4)

**Streams available (6):**
`PCMB`, `PCB`, `PCM`, `PCBZ`, `Commerce`, `Humanities`

**What it renders:**

**Recommended Stream Card:**
- Gradient card (light blue → light purple) with rounded corners
- Center-left: "Recommended Stream:" label, large bold stream name (28pt), full stream title
- Compatibility bar: `"Compatibility: [green gradient bar] XX%"`
- Right side: 2×2 grid of glass-circle department icons from `STREAM_SELECTION_CONTENT[stream].fields` (first 4 fields)

**Reasons section:**
- Bold "Reasons" heading
- Pill badges from `TRAIT_REASONS[traitCode]` — displayed in a horizontal flowing row, wrapping when needed
- Each pill has a green checkmark circle icon + reason text

**Alternative Streams:**
- 3 side-by-side gradient cards (purple-blue gradient)
- Each shows: stream short name + compatibility %, full stream title, progress bar

**Future Direction box:**
- Light box showing: `"Future direction"` header + `"Possible future pathways: [text from STREAM_FUTURE_DIRECTIONS[altStream]]"`

## 5. Core Computation Logic

### 5.1 DISC Trait Scoring & Sorting

```typescript
// Four raw scores from assessment (0–100)
const scores = [
  { type: 'D', val: score_D },
  { type: 'I', val: score_I },
  { type: 'S', val: score_S },
  { type: 'C', val: score_C },
];

// Sort descending by value; tie-break by priority order C > D > I > S
const PRIORITY = ['C', 'D', 'I', 'S'];
scores.sort((a, b) => {
  const diff = b.val - a.val;
  if (diff !== 0) return diff;
  return PRIORITY.indexOf(a.type) - PRIORITY.indexOf(b.type);
});

// ci_sortedTraits[0] = dominant, [1] = secondary, [2] and [3] = development zones
// ci_topTwo = e.g. "CD", "DI", "SC"
```

### 5.2 Profile Pattern Detection
**Method:** `ci_detectPatterns()`

```typescript
// discType
if (top.val > 75 && gap > 10)  → 'dominant'
if (gap < 5 && top.val > 65 && second.val > 65)  → 'dual'
if (stddev < 8)  → 'balanced'
if (gap >= 5)  → 'dominant'
else  → 'balanced'

// agilePattern (using normalized 0–100 agile values)
if (courage > respect + 20)  → 'assertive-risk'
if (respect > courage + 20)  → 'cautious-respect'
if (focus > 70 && commitment > 70)  → 'execution-engine'
if (openness > commitment + 20)  → 'creative-instability'
if (commitment > openness + 20)  → 'steady-execution'
else  → 'balanced'

// stressType
if (D > 70 && S < 50)  → 'assertive'
if (C > 70)  → 'overthink'
if (S > 70 && D < 40)  → 'withdrawal'
else  → 'balanced'

// academicStyle
if (C > 65 && focus > 65)  → 'structured'
if (I > 65 && openness > 65)  → 'collaborative'
if (S > 65 && commitment > 65)  → 'self-paced'
if (D > 65 && courage > 65)  → 'competitive'
else  → 'structured'

// textVariant (cycles 3 text strings)
textVariant = (D + I + S + C) % 3
```

### 5.3 Agile Score Normalization
```typescript
// Raw agile values are 0–25; normalized to 0–100 for comparisons
const norm = (v: number) => Math.min(100, Math.round((v / 25) * 100));
```

### 5.4 Career Fit Score Calculation
**Method:** `ci_getCareerFitScores()`

For each of 4 career models, a **weighted proximity score** is computed:
```typescript
weightedScore = targets.reduce((sum, dim) => {
  const closeness = Math.max(0, 100 - Math.abs(profileValue[dim.key] - dim.target));
  return sum + closeness * dim.weight;
}, 0);
```
The profile is compared against ideal target values for each dimension (D, I, S, C, Courage, Focus, Commitment, Openness, Respect). Closer to the ideal = higher closeness. Weights sum to ~1.0 per model.

**Confidence:**
- Gap between #1 and #2 ≥ 8% → `"High"`
- Gap ≥ 4% → `"Moderate"`
- Gap < 4% → `"Exploratory"`

Uses `most_answered_answer_type` array (frequency of answers per DISC type) as primary signal, with `score_X` as tiebreaker. Returns `[primary, secondary]` as e.g. `['S', 'D']`.
